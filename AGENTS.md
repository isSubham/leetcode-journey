# Agent context for this repo

This repo is an auto-organized archive of LeetCode solutions, sorted into folders by topic (see `config/tag-folder-map.json` for the mapping rules). Each problem folder under `solutions/<Topic>/<problem-slug>/` contains the original problem's `README.md` and a solution file in whatever language it was submitted in.

## How to help

When asked to analyze progress, you can:
- Count solution folders per topic to identify strong vs weak areas.
- Read solution files within a topic folder to spot repeated patterns, repeated mistakes, or inconsistent style across languages.
- Use `git log` on `solutions/` to build a timeline of what was learned when.
- Cross-reference `config/tag-folder-map.json` to understand which LeetCode topic tags map to which folder.

## Example prompts this repo supports

- "Look at my solutions/DP folder and tell me what patterns I've covered and what I'm missing."
- "Review my solutions/Graphs folder for style consistency across languages."
- "Based on solutions/, suggest my next 5 problems to close gaps."
- "How has my solving style changed over the last 3 months based on commit history?"

## What not to do

- Don't modify files inside `solutions/` unless explicitly asked — this folder is auto-generated by the sync workflow and manual edits will be overwritten on the next run.
- Don't assume every problem has a single canonical solution — some folders may contain multiple attempts if the sync ran before deduplication.
